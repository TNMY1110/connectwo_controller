# 06.09 | 하드웨어 매칭 및 통신 선로 개통

## ROS-STM32 시리얼 통신 불일치 해결
- **현상:** 라즈베리 파이의 수신 `BaudRate`가 `230400`으로 고정되어 있어, STM32와의 `BaudRate` 불일치로 인해 ROS 통신이 불가능했음

- **조치:** `motor_controller.ioc` 및 `usart.c`에서 `USART2 BaudRate`를 `230400`으로 상향 변경

## 로봇 물리 제원 동기화
실제 로봇 베이스 제원에 맞춰 구동 알고리즘 상수 수정
* **휠 베이스(바퀴 간격):** `250mm` -> **`260mm`** (AuCAR_conf.h)
* **바퀴 반지름:** `65mm` -> **`57mm`** (AuCAR_conf.h 및 connectwo_config.h)


## 메모리 최적화
- **현상:** 메모리 한계로 인해 STM32 내부의 `odom_to_tf` 및 `joint_state` 연산 불가

- **조치:** TF 변환 연산을 라즈베리 파이로 이관

# 06.10 | 제어 주기 동기화 및 틱 누적 구조 개편
## 하드웨어 타이머 주기 오류 수정
- **현상:** 기존 TIM7 설정이 `Period` = `3999`로 되어 있어 연산 주기가 $40\text{ms}$로 느리게 돌고 있었음. 이로 인해 상위 제어 주기와 싱크가 맞지 않음

- **조치:** `Period` = `999`로 수정하여 APB1 타이머 클럭(64MHz) 기준 정확히 10ms 제어 주기 동기화

## 엔코더 틱 유실 방지 알고리즘 도입

- **현상:** 기존 공식 코드의 `motorControl()` 내부에서 레지스터가 초기화(`*CNTx = 0`)되어 `50ms` 주기로 밖에서 읽을 때 값이 증발

- **조치:** `Motor.h`에 날것의 변위값을 반환하는 `getDeltaEncoderValue()` 구현 및 `accFactor` 상수를 1로 수정  
`timer10ms` 루틴 내에서 `10ms`마다 발생한 델타 틱을 전역 변수 `accumulated_left_tick` / `right_tick`에 실시간 누적하는 구조로 개편


# 06.11 | 엔코더 비대칭성 발견

## 하드웨어 엔코더의 치명적 기하학적 비대칭성 발견 (덤프 분석)

- **현상:** 바퀴를 직접 굴려 모터 카운트를 확인한 결과, 방향에 따라 분해능이 반토막 나고 부호가 뒤집히는 독특한 유격/노이즈 버그 확인

왼쪽 바퀴들 (`M0`, `M1`): 전진 시 `+1600` (4배수) / 후진 시 `-800` (2배수 탈락)  

오른쪽 바퀴들 (`M2`, `M3`): 전진 시 `-800` (2배수 탈락 및 부호 반전) / 후진 시 `+1600` (4배수 및 부호 반전)

음수/양수 틱이 서로 상쇄되어 ROS 오도메트리 상 전진 속도가 0.01 m/s 주변에 갇히던 원인 규명

- **조치:** `timer10ms` 내부에 삼항 연산자 기반 가변 필터를 설계하여, 2배수로 탈락하는 구간(`800`대)은 강제로 2배 연산하여 `1600`대로 맞추고 부호를 전진 방향(+)으로 일치시킴

- **결과:** `Rviz` 상에서 로봇이 실제 명령 속도에 맞춰 딜레이 없이 움직임
